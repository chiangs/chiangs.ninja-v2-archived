➊ When to SWITCH and when to IF/ELSE.

09.JAN.17

Both IF/ELSE and SWITCH are ways to compare variables. But which should you use and when?

Do you have multiple conditions and need to check through many logical decisions? IF/ELSE is the way to go. It allows you to quickly compare conditions that depend on user input.

Do you have a single variable you are switching from that makes it easy to jump directly to the right value? SWITCH is the construct you are looking for. SWITCH can be optimised by the compiler so you get 'faster' code depending on how long the list of checks are.

In essence, a SWITCH is really saying "pick one of these based on this variable's VALUE", but an IF/ELSE statement is a series of boolean checks of TRUE/FALSE and states that this portion of code depends on a number of possible variables rather than a single one.

➋ What is 'Math'? A '.random()' question.

10.JAN.17

In Java, 'Math' is a class of methods for performing basic numeric operations. 'Math', is capitalized because it is a static class of basic numeric functions for computation of primitives within the 'java.lang' package; it's static because there are no stand-alone functions in Java, so they all need to be a member of a class, and in many cases do not require it to be applied to an object.

When you append '.random()' to the Math class, it returns a double value with a positive sign, greater than or equal to 0.0 and less than 1.0. The result is random and can be used to add some random variability in methods when needed (e.g., a basic game of Rock, Paper, Scissors.).

public class RockPaperScissors {
    public static void main(String[] args) {

        rps = Math.random();
        if ( rps < 0.333333 ) { //will be true 1/3 of the time
        System.out.println( "Rock" );
        }
        else if ( rps < 0.6666667 ) {
        System.out.println( "Paper" );
        }
        else {
        System.out.println( "Scissors" );
        }
    }
}

➌ What kind of sorcery is this? 'Casting' a DOUBLE to an INT.

11.01.17

A DOUBLE and INT are both arithmetic variables, but different data types. An INT is more exact, as it requires the variable to be a whole number (e.g., 1, 24, 73, 99, etc.). An INT is a variable with decimals, or fraction, to denote a floating number (e.g., 1.2, 6.0., 99.999, etc.). They are also different in terms of number of bits used to store the variable's value.

When using a DOUBLE and INT together in Java, and many similar languages, it is often more convenient to use DOUBLES on calculations because of their flexibility or precision. However, sometimes after the computations are done, one might wish to simplify the result for the rest of the process. To do this, you cast the DOUBLE to an INT, simplifying the decimal number to a whole number.

In Java, there are several ways to do this:

Explicit Cast - (int) doubleValue; this will remove the decimal value of the double and keep the whole number but without rounding. In the example below, the .7 just drops resulting with INT value of 24.

public class DoubleToInt {
    public static void main(String[] args) {
        double doubleValue = 24.7;
        int intValue = (int) doubleValue;
        System.out.println("The int value is: " + intValue);
    }
}

Cast with Rounding - Math.round(doubleValue); Math.round accepts a double value to round it to the next whole number. Then we can cast it to an integer, resulting in an INT rounded up. In the example below, the result is an INT of 25.

public class DoubleToIntRound {
    public static void main(String[] args) {
        double doubleValue = 24.7;
        long longValue = Math.round(doubleValue);
        int intValue = (int) longValue;
        System.out.println("The int value is: " + intValue);
   }
}

Casting with - Double.intValue(); if the DOUBLE is an object of class 'Double', Double.intValue() can be used. In the example below, the result is an INT of 24.

public class DoubleToInt {
    public static void main(String[] args) {
        Double doubleObject = 24.7;
        int intValue = doubleObject.intValue();
        System.out.println("The int value is: " + intValue);
   }
}

➍ What is the difference between a ‘for’, ‘while’ and ‘do while’ loop?

12.JAN.12

Loops in Java, and many similar programming languages, are a set of decision-making statements. They check a statement, or a group of them, and execute the body of the loop while the condition is true. There are three types of loops:

  1. For loops
  2. While loops
  3. Do While loops

Each loop type differs in small but important ways. A 'for' loop iterates over a finite range of values until a particular condition is satisfied. It first checks the condition and if true, enters the loop until condition is no longer true. This is the most common type of loop when you have a list of items that you need to 'iterate' over for processing.

The 'while' loop is sometimes called a “pre-test” loop because it checks the condition first, while the 'do-while' is called a “post-test” loop because it checks the condition after running the body of the loop at least once. These two loops are great when you don't want to repeat code a number of times, rather, keep looping through the code until a certain condition is met (or in other words, keep looping while the condition is true). It looks like this:

  1. Check the 'while' loop condition, if false, don’t execute any code inside the block.
  2. If condition is true, execute code inside 'while' block
  3. Repeat step 1.

So why bother to make a do-while loop function when most often a 'while' loop is all that's required? Sometimes you need a 'do while' because what is being checked in the condition is something you don’t really know until you have gone through the body of the loop at least once.

  1. Executes the code in loop.
  2. Checks the condition, if true, continues to execute the loop.
  3. If false, exits the loop.

➎ Explaining the 'void'

23.JAN.17

Java programmers are very familiar with the main method, but it's common to forget what it really means. Here's what it looks like:

    public class SomeName {
        public static void main(String args) {

        }
    }


In a nutshell, the main method 'signature' can probably be thought of as providing all the information there is about the interface to the method; it provides all the information that you need to know to be able to invoke the method.

  • Main is 'public' so that its visible to every other class, even to which are not part of its package.
  • Main is 'static' in Java so that it is indicated as a class method, so that it can be called without creating any instance.
  • Main is 'void' because the method itself doesn't return any value, but the return type must be indicated even if 'nothing'.
